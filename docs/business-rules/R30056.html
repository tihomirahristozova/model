R30056 PosOperator - Users Person Required

    Code R30056
    Entity Crm.Pos.Operators Entity
    Name UsersPersonRequired
    Attribute User
    Layer Back-End
    Events Commit
    Priority Early
    Modify NO
    Applicable Legislations BG (PosLocation.EnterpriseCompany.Company)
    Action Validate that:

    User.Person != NULL
    Description The current rule reassures that the POS Operator is associated with a particular Person.

    For this reason, the rule validates that the Person field is filled into the definition of the User that is set for the POS Operator.
    Message POS Operator with a code '{PosOperatorCode}' contains a user '{User.Login}' which is not associated with a Person. Please, make sure that the Person field in the User's definition is filled in.
    Version Introduced: 2019.1
    Revocable NO
