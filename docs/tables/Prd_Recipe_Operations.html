    Table Prd_Recipe_Operations

    Entity

    Entity: Production.Technologies.RecipeOperations

    Contains the routing (operation list) of the recipes. Entity: Prd_Recipe_Operations

    Owner Tables Hierarchy

    • Prd_Recipes

    Summary

    Name Type Description
    Id uniqueidentifier PK
    Line_Ord int Order of the operation within the recipe
    Minimum_Concurrent_Start_Time_Minutes int How many minutes after the start of this operation can the next operation start. NULL means that the next operation should wait this operation to finish before starting
    Move_Time_Minutes int Time to move the lot to the next operation in minutes
    Notes nvarchar(254)
    Operation_Description nvarchar(max) The description of the operation.
    Operation_Id uniqueidentifier Standard operation Id. If not NULL used to load the details. If NULL the details (times, scrap rates, etc.) must be entered manually
    Operation_Instruction_Id uniqueidentifier Link to additional data, containing instructions in external format
    Recipe_Id uniqueidentifier
    Routing_Operation_Id uniqueidentifier
    Row_Version timestamp
    Run_Time_Minutes int Duration of the operation for standard lot of the product
    Scrap_Rate decimal(7, 6) Standard rate of scrap during the operation
    Setup_Time_Minutes int Time needed to setup the equipment
    Standard_Cost_Per_Hour decimal(18, 6) Standard cost per hour for this operation. It participates in the calculation of standard cost for production for the whole recipe.
    Standard_Price_Per_Hour decimal(18, 6) Standard price for 1 hour work
    Tooling nvarchar(max) The tools needed for the routing step
    Use_Quantity decimal(9, 3) Quantity of the workgroup resource that should be allocated for the operation
    Use_Quantity_Unit_Id uniqueidentifier The measurement unit of Use_Quantity
    Wait_Time_Minutes int Wait time (drying, cooling, etc.) after the operation in minutes
    Workgroup_Resource_Id uniqueidentifier The exact workgroup resource that is engaged in the operation. NULL means that no resource is needed or it will be specfied at a later stage
