    Table Sec_Domain_Providers

    Entity

    Entity: Systems.Security.DomainProviders

    The external authentication providers, linked to the domain. Contains data about registration, usually the app id or app key. Entity: Sec_Domain_Providers (Introduced in version 20.1)

    Owner Tables Hierarchy

    • Sec_Domains

    Summary

    Name Type Description
    __Object_Version int
    Client_ID nvarchar(254) This is the Client_ID of our registration within the provider.
    Client_Secret nvarchar(254) This is the password, which we, as a client app, use to access the authentication provider.
    Display_Name nvarchar(254) ML Specifies the (multi-language) display name of the button in the login form. Especially useful for multi-tenant providers.
    Domain_Id uniqueidentifier The domain for which the provider is setup.
    Domain_Provider_Id uniqueidentifier PK
    Is_Active bit Specifies whether the provider is currently active for the domain.
    Notes nvarchar(max)
    Provider_Name nvarchar(256) Allowed: ERPNET, AZUREAD, GOOGLE, FACEBOOK The system name of the user authentication provider. For example GOOGLE, FACEBOOK, MICROSOFT, AZUREAD, ERPNET, etc.
    Row_Version timestamp
    Tenant_ID nvarchar(254) When the authentication provider is multi-tenant, this specifies the tenant ID. For example, Google, Facebook and Microsoft are singe-tenant, while AzureAD is multi-tenant.
